项目一 算法与程序基础

任务一 算法的定义及特性

算法定义

广义

算法是指在解决问题时，按照某种机械的步骤一定可以得到问题的结果（有解时给出问题的解，无解时给出无解的结论）的处理过程。

狭义

算法就是计算机解决问题的过程。

算法三要素

操作

  1. 算数运算：加（+）、减（-）、乘（*）、除（/）。
  2. 关系比较：大于（>）、小于（<）、等于（=）、不等于（!=）
  3. 逻辑运算：与（and）、或（or）、非（not）。
  4. 数据传送：输入、输出、赋值（计算）。

控制结构

各操作之间的执行顺序，即控制结构。

  1. 顺序结构：各操作依次进行。
  2. 选择结构：由条件是否成立来决定选择执行。
  3. 循环结构：有些操作需要重复执行，直到满足某个条件时才结束，这种控制结构也称为重复或者迭代结构。

数据结构

算法操作的对象是数据，数据间的逻辑关系、数据的存储方式及处理方式就是数据结构。

算法的基本性质

  1. 目的性
  2. 分步性
  3. 有序性
  4. 有限性
  5. 操作性

任务二 算法的描述与设计

子任务一 算法的描述

算法就是解决问题的方法和步骤，而且步骤是有限的。

描述方法

  • 自然语言描述法
  • 流程图描述法
  • N-S描述法（盒图）

流程图

流程图是算法的图形化描述语言，流程图可以清晰的描述算法的思路和过程。

常用的流程图符号

子任务二 算法设计

算法设计的要求

  1. 正确性
  2. 可读性
  3. 稳健性（当输入非法数据时，给出相应的处理而不是输出某明奇妙的结果）
  4. 高效率与低存储量（高效率指的是算法的执行时间，低存储量指的是算法执行过程中所需的最大存储空间）

算法的重要特性

  1. 有穷性
  2. 确定性
  3. 可行性
  4. 算法有零个或多个输入
  5. 算法有一个或多个输出

任务三 算法分析和评价

子任务一 算法的时间复杂度

时间复杂度定义

算法的时间复杂度是衡量算法执行时间随输入规模增长而不断变化的度量。

作用

它用来估计算法在最坏情况下（该算法最多执行次数，通常带有条件判断）所需的时间资源。

时间复杂度相关因素

  1. 问题中数据存储的数据结构。
  2. 算法采用的数据模型。
  3. 算法设计的策略。
  4. 问题的规模。
  5. 实现算法的程序语言。
  6. 算法编译产生的机器代码质量。
  7. 计算机执行指令的速度。

算法的时间复杂度

时间复杂度不是一个明确的执行时间，而是一个函数。时间复杂度通常使用符号 “ O ”。

常见分类
  1. 常数时间复杂度（ O(1)）
  2. 线性时间复杂度（ O(n) ）
  3. 对数时间复杂度（ O(log n) ）
  4. 平方时间复杂度（ O(n^2) ）
  5. 指数时间复杂度（ O(2^n) ）

子任务三 算法的空间复杂度

空间复杂度的定义

一个算法空间复杂度S（n）定义为该算法所耗费的存储空间

作用

它是对一个算法在运行过程中临时占用存储空间大小的量度。

常见分类

  1. 存储算法本身所占用的存储空间。
  2. 算法的输入输出数据临时占用的存储空间。
  3. 算法在运行过程中临时占用的存储空间。

时间复杂度和空间复杂度的关系

对于一个算法**时间复杂度和空间复杂度往往是相互影响的，当设计一个算法时要综合考虑**时间复杂度和空间复杂度

任务四 算法与程序的关系

算法不等于程序，程序是算法用某种程序设计语言的具体实现，程序不一定满足算法的五大特征（比如操作系统就是一个无限循环执行的程序，不满足有限性）

算法是解决问题的方法，而程序是则是算法的具体实现。

算法的好坏直接影响程序的**效率和质量**。

程序的好坏取决于算法的好坏和编程人员的水平。

算法是程序的灵魂。

任务五 典型算法举例

子任务一 排序算法

1.选择排序

是最基础的算法。

原理

假设第一个元素是最小值，然后依次与后面的元素作比较，如果有更小的值，就让其交换位置，以此类推，直到最后完成排序。

流程图

2.冒泡排序

原理

从第一个元素开始，依次对相邻的两个元素进行比较，如果前一个元素大于后一个元素则交换它们的位置，以此类推直到最后一个元素为止

流程图

子任务二 查找算法

1.顺序查找法

顺序查找也称线性搜索，在数组中找出与给定关键字相同的值的具体位置。

原理

让关键字与队列中的第1个（或最后1个）位置的值逐个比较，直到找出与给定关键字相同的值为止。它的缺点是效率低下。

流程图

2.二分查找法

二分查找是一种高效的搜索算法,适用于有序数组。它通过不断将搜索范围缩小一半来查找目标值,直到找到目标值或确定目标值不存在。

注：适用于且只适用于有序数组。

原理

假设我们要在一个有序数组中查找目标值。我们首先将搜索范围设定为整个数组,然后找出数组中间位置的元素。如果该元素等于目标值,则查找成功;如果该元素大于目标值,则目标值可能在数组的前半部分;如果该元素小于目标值,则目标值可能在数组的后半部分。不断重复上述步骤直到找到目标值或确定目标值不存在。
